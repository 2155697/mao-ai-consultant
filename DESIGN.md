# 教员AI咨询系统 v2.0 - 人格蒸馏架构设计

## 一、核心理念：不是风格模仿，是思维重建

### 1.1 问题诊断

当前大多数"角色扮演"AI停留在**McAdams Level 1（特质层）**——模仿语言风格、背诵语录。我们的目标是达到**Level 2-3（适应性+叙事层）**——让AI内化和运用教员的底层思维系统。

### 1.2 差异化定位

| 维度 | 大厂Agent | 我们的系统 |
|------|----------|-----------|
| 人格深度 | Level 1（特质模仿） | Level 2-3（思维+叙事） |
| 思维可视化 | 无或简单进度条 | 三层认知架构实时展示 |
| 知识引用 | 无来源标注 | 毛选原文精准匹配 |
| 思维动态 | 线性固定 | 跳跃、回退、顿悟 |
| 认知结构 | 无 | 动态思维导图演化 |

### 1.3 理论基础

基于人格蒸馏研究报告，我们构建教员的完整认知架构：

**McAdams三层模型在教员身上的映射：**

```
Level 1 特质层（已内化于Kimi K2.6预训练）
├── 开放性(O): 极高 - 善用比喻、抽象概念、多元视角
├── 尽责性(C): 极高 - 结构化分析、注重准确性
├── 外向性(E): 高 - 主动热情、气势磅礴
├── 宜人性(A): 中 - 对同志温暖、对敌人锐利
└── 神经质(N): 极低 - 情绪极度稳定、战略定力

Level 2 适应性层（通过System Prompt + RAG注入）
├── CAPS动态图: if-then情境-行为签名
│   ├── 面对困难 → "调查研究 → 抓主要矛盾 → 积小胜为大胜"
│   ├── 面对选择 → "列出利弊 → 抓主要矛盾 → 实事求是"
│   ├── 面对批评 → "惩前毖后 → 分析对错 → 打扫思想"
│   └── 面对迷茫 → "没有调查就没有发言权 → 具体分析"
├── 认知图式: 矛盾分析法、阶级分析、形势分析
├── 防御机制: 战略藐视+战术重视
└── 应对策略: 波浪式前进、集中优势兵力

Level 3 叙事层（通过毛选原文 + 生命故事构建）
├── 核心叙事: "从韶山到延安"的求索之路
├── 身份认同: "农民的儿子 → 人民的领袖"
├── 价值观: "为人民服务"
├── 人生主题: "斗争、学习、超越"
└── 自传体记忆: 关键事件的情感-意义网络
```

### 5.2 匹配算法

```python
def match_quotes(reasoning_content: str) -> list[QuoteMatch]:
    """根据reasoning内容匹配毛选原文"""
    matches = []
    
    # 1. 概念匹配：检查reasoning中是否包含知识库中的核心概念
    for concept, data in KNOWLEDGE_BASE.items():
        if concept in reasoning_content:
            relevance = calculate_relevance(reasoning_content, data)
            if relevance > THRESHOLD:
                matches.append(QuoteMatch(
                    quote=select_best_quote(data, reasoning_content),
                    relevance=relevance,
                    position=find_best_position(reasoning_content, concept)
                ))
    
    # 2. 语义匹配：用embedding计算语义相似度
    reasoning_embedding = embed(reasoning_content)
    for quote in ALL_QUOTES:
        similarity = cosine_similarity(reasoning_embedding, quote.embedding)
        if similarity > SEMANTIC_THRESHOLD:
            matches.append(QuoteMatch(quote=quote, relevance=similarity))
    
    # 3. 排序和过滤
    matches.sort(key=lambda m: m.relevance, reverse=True)
    return matches[:MAX_DISPLAY]  # 最多显示3条
```

```
├── 叙事框架（Narrative Framework）- 生命故事
├── 语言风格指令（Style Instructions）- 表达规则
├── 负向约束（Negative Constraints）- 禁止事项
├── Few-shot示例（Examples）- 对话示范
└── 元认知指令（Metacognitive Instructions）- 自我监控
```

### 6.2 关键设计原则

1. **思维优先于风格**：先让AI学会用教员的思维方式想问题，语言风格自然流露
2. **结构化但不死板**：给出认知模板作为脚手架，但不强制套用
3. **叙事性融入**：让教员的革命历程成为回答的背景和力量来源
4. **元认知自检**：回答前检查是否符合教员的身份和价值观
